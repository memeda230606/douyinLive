# DouyinLive Desktop 用户指南

本文适用于 `0.2.x` Windows 10/11 x64 版本。程序用于管理用户有权观察的直播间，在本机等待开播、保存互动事件和媒体，并提供历史回放、基础分析、导出与受签名保护的自动更新。

## 安装与首次启动

1. 只从项目维护者提供的可信渠道取得安装包，并先按同版本 `release-manifest.json` 核对 SHA-256。
2. 当前直接/内部交付不要求 Authenticode 签名，Windows 可能显示 SmartScreen 提示。只有来源可信且 SHA-256 完全匹配时才继续运行；公开分发时仍建议使用带可信时间戳的代码签名。
3. 安装器按当前用户安装，不需要管理员权限。缺少 WebView2 时会自动静默运行包内锁定的 Microsoft 官方 Evergreen Bootstrapper，安装成功后继续部署。
4. 全新安装首次启动时必须在引导窗口确认录制目录；可使用 Windows 系统目录选择器，也可输入绝对路径。程序会创建目录并验证可写性，确认后才允许新录制。之后可在“设置 → 存储与录制”随时修改；修改只影响之后启动的录制，不移动或中断正在录制的场次。

## 添加直播间与监控

1. 在“直播间”添加 `live.douyin.com/<id>` 的路径标识，可设置本地显示名称和录制策略。
2. Cookie 是可选项，仅在确有授权和必要时提供。保存后前端不会回显原值。
3. 启用监控后，离线房间会持续等待；检测到开播后建立消息连接，并按录制策略选择流。
4. 总览和实时页面展示连接、录制、重试、缺口及磁盘告警。遇到错误时记录稳定错误码，诊断信息不包含完整流地址或 Cookie。

默认只允许一个房间同时录制。提高并发上限前应确认磁盘吞吐、剩余空间和 CPU 余量；等待开播和消息监听可以并行。

## 直播结束与历史复盘

- 下播、人工停止或不可恢复错误后，程序会收尾分片、探测媒体、生成代理文件并更新场次清单。
- “历史”页面按场次分页显示媒体、互动事件和已知缺口；只有通过完整性与身份校验的 MP4 才直接播放。
- 时间线、指标桶、峰值、低谷和高光候选使用统一毫秒偏移。缺口会降低完整度，不会被插值成完整数据。
- 默认 ASR 提供器为 disabled；未配置转写时，基础分析仍可完整生成。

## 导出与诊断

- 分析导出生成版本化 JSON manifest 与 CSV。默认排除昵称、弹幕正文、转写正文、平台标识、媒体路径、Cookie、签名和完整 URL。
- 显式选择正文导出前应确认接收方和保存位置；CSV 会处理电子表格公式注入，但导出内容仍可能包含用户主动选择的文本。
- 诊断包由用户主动生成和发送。程序不默认上传遥测、日志、媒体或业务数据。
- 诊断或导出失败时保留稳定错误码，不要把原始数据库、凭据文件或完整直播地址发送到公开问题区。

## 自动更新

- 设置中的“自动更新”默认开启。程序启动成功 30 秒后检查一次，之后约每 6 小时检查；关闭后不再周期联网，但仍可点击“立即检查”。
- 更新清单和安装器来自固定阿里云 OSS HTTPS Origin。客户端先验证内置 Ed25519 公钥签名，再验证版本、平台、对象前缀、大小和 SHA-256；本地不保存 OSS 或发布凭据。
- 检查到新版本可以查看纯文本发布说明。只有全部直播间不处于启动、直播、录制、重连或收尾状态时才会下载；安装始终需要点击“安装并重启”确认。
- 下载支持 ETag 与 Range 断点续传。签名、大小、来源或 SHA-256 不匹配时会删除临时文件并显示稳定错误码。
- 安装前程序创建 SQLite 一致性备份并检查系统盘和数据盘空间。独立更新助手等待程序正常退出、运行新安装器并等待新版本健康标记；失败时恢复旧程序目录，并在 schema 已变化时使用精确升级前备份恢复数据库。
- 如果数据库回滚失败，更新助手不会启动可能不兼容的旧版本，也不会删除备份；请保留错误码和备份目录，按人工恢复说明处理。
- 已安装版本不会自动降级。通道回退只影响尚未升级的客户端，坏版本必须由更高 SemVer 修复。

## 备份、升级和回滚

- 数据位于 `%LOCALAPPDATA%\DouyinLive`，程序文件位于当前用户安装目录，二者相互独立。
- 升级前先退出程序，并备份重要外部媒体。数据库迁移前会创建一致性备份；迁移失败会停止启动。
- 降级程序不能直接打开较新的 schema。需要回滚时使用同版本安装目录内的 `douyin-live-dbrollback.exe` 和升级前备份，严格步骤见 `INSTALLATION.md`。
- 不要手工覆盖 `app.db`、删除活动的 `-wal`/`-shm`，或在程序运行时移动内部数据根。

## 卸载与删除数据

- 默认卸载只删除程序、快捷方式和 WebView2 缓存，保留数据库、内部媒体、配置、日志、导出和备份。
- 业务数据只在交互式卸载中主动选择删除并完成第二次确认后清理；静默卸载永远保留数据。
- 外部媒体目录始终由用户管理，卸载器不会删除。

## 故障处理

1. WebView2 自动安装失败：确认可访问 Microsoft 下载服务后重试；离线环境可先安装 Microsoft 官方 x64 Evergreen Standalone Installer。
2. FFmpeg 校验失败：不要替换随附二进制；重新下载并核对正式发布清单。
3. 磁盘不足：释放目标数据盘空间或更换经过验证的外部媒体目录；消息监听会尽量继续，录制可能停止。
4. 数据库忙或异常退出：保留数据根并重启；程序会按耐久 checkpoint 恢复。不要同时启动多个实例操作同一数据根。
5. 平台连接或解析失败：查看脱敏错误码并有界重试。平台字段、风控和网络变化可能需要后续兼容更新。

正式发布前还应阅读同包的 `PRIVACY.md`、`KNOWN-LIMITATIONS.md`、`RELEASE-CHECKLIST.md` 和第三方许可证材料。
