# 0.1.x 已知限制

## 平台与数据完整性

- 平台字段、签名、风控和直播流地址可能变化；程序不能保证 100% 获取弹幕、礼物、点赞、进场或媒体。
- 产品不提供直播后台专属的曝光来源、个人观看时长、商品点击或订单转化数据。
- 断网、流过期、进程异常和磁盘不足会形成显式缺口。基础分析会报告完整度，不会把缺口推断为真实互动。
- 默认只允许一个房间同时录制；提高并发上限需要用户自行评估磁盘与 CPU。

## 回放与分析

- 只有身份、大小、SHA-256、容器和编码均通过校验的 H.264 MP4 才直接播放；其他媒体可能只显示状态或需要后续转码。
- 相邻且已验证的 MP4 支持续播；真实缺口、时间戳异常或损坏分片不会被无缝隐藏。
- 峰值、低谷和高光是版本化规则产生的候选，不是业务因果结论。
- ASR 默认 disabled，0.1.x 不随附本地模型或云端提供器；没有转写时基础分析仍可用。

## 系统与安装

- 首发安装包只支持原生 Windows x64，并依赖 Microsoft Edge WebView2 Evergreen Runtime。
- arm64 没有独立构建和安装矩阵；Windows Server、Wine、虚拟化兼容层和非原生架构不在首版承诺范围。
- 当前本地正式验收主机为 Windows 11 x64 build 22631；Windows 10 实机最终安装/运行证据在取得前必须保持发布阻塞。
- 开发候选可能未签名。只有 Authenticode 状态有效、时间戳有效且校验和写入最终清单的包才可称为正式发布候选。

## 病毒扫描与供应链

- 本地验收主机由第三方杀毒接管，Microsoft Defender 服务处于停用状态；没有可复核扫描报告时不得把“未发现告警”等同于病毒扫描通过。
- 单一引擎扫描不能替代发布所需的受控杀毒矩阵。扫描报告必须绑定最终签名后文件或最终发布清单的 SHA-256。
- FFmpeg 8.1.2 Essentials 随包分发并采用 GPL-3.0-or-later；完整组件、许可证和 notices 以同版本 SBOM/许可证文件为准。
- 一个未携带许可证声明的间接 Go 模块在 SBOM 中保留为 `NOASSERTION`，没有猜测许可证。

## 恢复、卸载与数据

- 数据库 schema 不自动降级；旧程序只能配合升级前备份和离线回滚工具使用。
- 默认卸载保留本地数据，静默卸载永远保留数据。交互式删除需要第二次确认且不可恢复。
- 外部媒体目录由用户管理；移动、替换或重解析活动数据根可能被安全门禁拒绝。
- 异常退出后的恢复以耐久 checkpoint、数据库状态和文件身份作为事实源；无法证明安全时会失败关闭并保留证据。

## 非目标

0.1.x 不提供云账号、团队协作、远程控制、移动端、浏览器 SaaS、自动发弹幕、自动发布视频或主播账号控制。高级大模型总结和跨场次智能建议不是首版发布门槛。
