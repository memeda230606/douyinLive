# 隐私说明

DouyinLive Desktop 是本地优先的直播记录与复盘工具。用户应只监控自己管理或已获授权的直播间，并自行遵守平台规则、当地法律及对直播参与者的告知义务。

## 本地保存的数据

根据启用的功能，程序可能在本机保存：

- 直播间配置、本地显示名称和监控策略；
- 场次、互动事件、聚合指标、已知缺口和恢复 checkpoint；
- FFmpeg 媒体分片、音频/播放代理和校验信息；
- 本地诊断日志、数据库备份、分析报告和用户导出；
- 可选 Cookie 的加密引用。

默认内部数据根为 `%LOCALAPPDATA%\DouyinLive`。用户选择的外部媒体目录不由卸载器管理。

## 凭据与平台数据

- Cookie 仅在用户主动配置时使用，并通过 Windows DPAPI 保护；前端、Wails DTO、日志和诊断摘要不回显原值。
- 完整流 URL、签名参数、`msToken`、`a_bogus`、原始 Protobuf 和平台原始标识不会进入前端状态或普通诊断导出。
- 用户标识在标准化数据中使用安装级 HMAC；显示名称和正文受隐私设置及导出选项控制。
- 平台响应和媒体地址可能短时有效。程序不会把真实响应、Cookie 或有效 URL 写入项目 fixture、发布文档或提交历史。

## 网络与第三方服务

程序为等待开播、消息连接和媒体录制访问抖音相关接口；WebView2 缺失提示只打开 Microsoft 官方运行时页面。基础版本不默认上传遥测、崩溃报告、诊断包、数据库、媒体或导出。

ASR 接口默认禁用，没有内置云转写提供器。未来配置第三方 ASR 时，应单独阅读该提供器的隐私条款；基础分析不依赖 ASR。

## 日志、诊断和导出

- JSONL 日志执行 URL、凭据、路径与敏感键脱敏并按保留策略轮转。
- 诊断包默认排除凭据、完整 URL、签名、原始 payload、视频和完整弹幕正文；脱敏后再次扫描，命中敏感模式则拒绝发布。
- 分析导出默认排除昵称、正文、平台标识、路径和摘要。只有用户显式选择时才导出允许的正文列。
- 导出文件离开本机后的保管、分享和删除由用户负责。

## 保留、备份与删除

用户可通过应用保留策略清理本地数据，也可在交互式卸载中经过第二次确认删除内部数据。默认卸载保留全部业务数据，静默卸载不提供删除开关，外部媒体永不自动删除。

数据库迁移会创建本地一致性备份；成功回滚后仍保留被替换数据库，避免自动销毁恢复证据。删除备份前应确认旧版本和数据完整。

## 用户责任与边界

本工具不能保证获取全部平台消息，也不应被用于绕过访问控制、采集无权处理的数据、自动发送内容或控制主播账号。若直播参与者、客户或团队政策要求更严格的保留期限、访问控制或加密，应在使用前配置并验证相应流程。
